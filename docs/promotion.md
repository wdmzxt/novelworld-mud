# NovelWorld MUD Promotion

## 一句话 Slogan

把小说变成可游玩的双语文字冒险世界。

## 中文宣传文案

NovelWorld MUD 是一个本地优先的双语文字冒险 MUD 引擎，目标是把小说转化为可以探索、对话、拾取物品、完成任务和触发事件的互动世界。

当前 v0.8 已经支持中文/英文双语显示、JSON 存档、任务系统、事件系统、数据驱动世界配置和 World Schema 校验。它暂时不接入 AI，不依赖数据库，也不需要 API Key，重点先把一个稳定、可理解、可扩展的本地 MUD 骨架跑通。

## English Copy

NovelWorld MUD is a local-first bilingual MUD engine for turning novels into playable text adventure worlds.

The current v0.8 release supports Chinese/English presentation, JSON save/load, quests, events, data-driven world content, and world schema validation. It does not use AI, databases, or API keys yet. The focus is a stable, inspectable, local text adventure foundation that can later support novel-to-world generation and optional LLM narration.

## 知乎 / V2EX 发布版

我在做一个小项目：NovelWorld MUD。

它的目标是把小说转成可玩的文字冒险世界。现在 v0.8 已经实现了一个本地命令行 MUD 原型，支持中文/英文双语、JSON 存档、任务、事件、数据驱动世界配置和基础 schema 校验。

目前刻意没有接入 AI，也没有用 SQLite / ChromaDB。原因是我想先把“小说世界应该如何结构化”和“基础 MUD 规则如何稳定运行”打牢。后续再考虑可选 LLM 叙事、小说导入和本地记忆。

Demo 世界是“青石村”：玩家可以在村口、村中小路、村长家、药田、后山、森林和山洞入口之间移动，和 NPC 对话，接任务，触发事件，并保存进度。

## Reddit 发布版

I’m building NovelWorld MUD, a local-first bilingual MUD engine for turning novels into playable text adventure worlds.

The current v0.8 prototype runs fully in the terminal and supports Chinese/English text, JSON save/load, quests, events, data-driven world content, and schema validation. There is no AI integration yet, no database, and no API key requirement.

The idea is to first build a stable rule-based foundation for representing fictional worlds, then later experiment with optional LLM narration and novel-to-world generation.
